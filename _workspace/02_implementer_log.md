## Verify-hook decision: docs-only stdio probe (NOT `--version`)
`n8n-mcp --version` is NOT a real version flag. Driving it boots the stdio MCP server and exits
0 only because stdin closes — it never prints a version, so it is an unreliable, falsely-passing
probe. Evidence:
```
$ printf "" | npx -y n8n-mcp --version
stdin closed, shutting down...
Received STDIN_CLOSE, shutting down gracefully...
rc=0          # exits 0 from pipe close, prints NO version — unreliable
```
Chosen instead: a minimal MCP `initialize` JSON-RPC handshake over stdio, asserting the
documentation-server identity. Key-less, fail-closed, never touches n8n or the key. Evidence:
```
$ printf '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{...}}\n' | npx -y n8n-mcp
{"result":{"protocolVersion":"2024-11-05","capabilities":{"tools":{},"resources":{}},
 "serverInfo":{"name":"n8n-documentation-mcp","version":"2.57.1"}},"jsonrpc":"2.0","id":1}

$ ... | grep -q 'n8n-documentation-mcp'; echo verify_rc=$?
verify_rc=0
```
The hook asserts `grep -q 'n8n-documentation-mcp'` → 0 only when the docs-only server actually
answers; non-zero if the bin is missing or doesn't respond. An absent key neither falsely
passes nor falsely fails it.
