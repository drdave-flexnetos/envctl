## Summary
`envctl doctor` / `auto-detect --json` falsely report the JS runtime as drifting because `node-via-bun`'s detect hook fails (real node `~/.local/bin/node` precedes the bun symlink on PATH) and its verify hook (`node --version`) can never pass on Bun by design. The fix is a pure-manifest change: drop the `group-ai-clis → node-via-bun` edge (so the fail-closed reverse-dep guard stops protecting it and nothing cascades), add a small `node-real` component that owns the real-Node-for-n8n requirement, and rewrite `node-via-bun`'s detect/verify hooks to tell the truth (detect succeeds when EITHER a bun node-symlink OR a real in-range node is present; verify uses a check Bun can actually satisfy). Zero engine/Rust code change; bun stays the default runtime; real node v22 is retained as the narrow n8n carve-out.

## Placement

- **`manifest/ai-clis.toml`** — drop `node-via-bun` from `group-ai-clis.requires`.
- **`manifest/envctl.lock`** — regenerated by `envctl lock` (49 → 50 components).
- **Engine-first rationale:** N/A — the engine is already correctly data-driven; the bug lives entirely in manifest data, not in `crates/engine`. The single most important finding: **no engine change is needed** (and none should be made).

## Work breakdown (leaf-first)

### Step 1 — `manifest/base.toml`: rewrite `node-via-bun` detect + verify hooks
Replace the current detect (lines 69-74) and verify (lines 79-82) bodies. Keep `id`, `name`, `requires=["bun"]`, `install`, `fix`, `remove` exactly as-is.

Detect — succeed when EITHER a bun node-symlink exists OR a real `node` is on PATH:
```toml
[component.detect]
kind = "command"
command = "bash"
# Truthful: node-compat is satisfied if EITHER ~/.bun/bin/node points at bun
# (the bun node shim) OR a real `node` resolves on PATH. On an n8n box the real
# node (~/.local/bin/node) precedes the inert bun symlink — that still counts.
args = ["-lc", "{ [ -x \"$HOME/.bun/bin/node\" ] && readlink -f \"$HOME/.bun/bin/node\" 2>/dev/null | grep -q bun; } || command -v node >/dev/null"]
```

Verify — use a node-compat check Bun CAN satisfy (`node -e`), never `node --version`:
```toml
[component.verify]
kind = "command"
command = "bash"
# Bun's node shim runs `node <script>` but NOT `node --version`. A real node also
# runs `node -e`. So `node -e` is the honest node-compat smoke test for BOTH
# runtimes. Prefer a real node if one is on PATH; else fall back to the bun shim.
args = ["-lc", "export PATH=\"$HOME/.bun/bin:$PATH\"; node -e 'process.exit(0)'"]
```

### Step 2 — `manifest/base.toml`: add a `node-real` component
Place immediately after the `node-via-bun` block, before `rustup`:
```toml
[[component]]
id = "node-real"
name = "Real Node.js (n8n V8 runtime)"
description = "Real Node 20-24 for tools that need V8 (n8n's isolated-vm). Bun (JSC) cannot satisfy these; this is the narrow non-bun carve-out. Bun remains the default JS runtime."
[component.detect]
kind = "command"
command = "bash"
# A REAL node (not the bun shim) in n8n's supported range 20-24.
args = ["-lc", "command -v node >/dev/null && ! readlink -f \"$(command -v node)\" 2>/dev/null | grep -q bun && node -e 'const m=+process.versions.node.split(\".\")[0]; process.exit(m>=20 && m<=24 ? 0 : 1)'"]
[component.verify]
kind = "command"
command = "bash"
args = ["-lc", "node -e 'const m=+process.versions.node.split(\".\")[0]; process.exit(m>=20 && m<=24 ? 0 : 1)'"]
[component.install]
kind = "script"
# Install a real Node 22 LTS into ~/.local without touching bun. nvm-free, no sudo.
script = '''
set -e
VER=v22.22.3
ARCH="$(uname -m)"; case "$ARCH" in x86_64) A=x64;; aarch64) A=arm64;; *) A="$ARCH";; esac
cd "$(mktemp -d)"
curl -fsSL -o node.tar.xz "https://nodejs.org/dist/$VER/node-$VER-linux-$A.tar.xz"
mkdir -p "$HOME/.local"
tar -xJf node.tar.xz --strip-components=1 -C "$HOME/.local"
'''
[component.fix]
kind = "script"
script = '''
set -e
VER=v22.22.3
ARCH="$(uname -m)"; case "$ARCH" in x86_64) A=x64;; aarch64) A=arm64;; *) A="$ARCH";; esac
cd "$(mktemp -d)"
curl -fsSL -o node.tar.xz "https://nodejs.org/dist/$VER/node-$VER-linux-$A.tar.xz"
mkdir -p "$HOME/.local"
tar -xJf node.tar.xz --strip-components=1 -C "$HOME/.local"
'''
# NOTE: no `remove` hook by design — n8n depends on real node; removal is not a
# managed operation here (avoids a footgun that would break n8n).
[component.wiring]
path_entries = ["~/.local/bin"]
```
*Implementer note:* `~/.local/bin` is already on PATH via the `bun` wiring; declaring it here too is idempotent. Real node currently at `~/.local/bin/node` so detect reads green immediately.
