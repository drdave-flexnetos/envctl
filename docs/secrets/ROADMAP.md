Eight phases, mirroring envctl's ROADMAP idiom. **Phase 0 compiles green** and ships the fail-closed `SecGuard` engine up front (real signatures + refusal types, `todo!()` bodies) â€” the safety-critical surface is never deferred. Each phase lists scope + acceptance criteria. The engine is a pure non-printing library emitting `SecretEvent`; `EngineError` is setup-time-only; operational outcomes are best-effort events, but the tamper-evident `audit_log` is written DURABLY.

> **Phase -1 blocker â€” RESOLVED (NEW-3, see `SERVER-MODE.md`):** OI-1 is ruled = **libSQL** (its server/replica/sync serves remote clients; redb is local-only and cannot). The bundled C SQLite is a SCOPED waiver: libSQL is quarantined in a NEW `secrets-store-libsql` crate consumed ONLY by `secretd` behind the `Store` trait; the engine stays pure-Rust and the no-C gate is per-crate. Phase 0 still targets the `Store` trait + `inmem-store`; **Phase 1 picks up the libSQL backend in the store crate, C-isolated** (recommended: embedded `sqld` on loopback + secretd's pure-Rust `remote` client). A new **Phase 8 (SERVER-MODE)** adds the remote relay HTTPS plane (in-process TLS edge, DPoP-bound bearers, control proven unreachable); the VPS profile is gated behind the operator-authorizer protocol + trusted-time open items. **Phase 8 status:** the `decide()` remote-binding ENFORCEMENT core (audit F3/F4 â€” `RemotePeer`, `CanonRequest.remote`, `VerifiedBearer.client_id/dpop_jkt`, cross-kind-presentation deny, DPoP fail-closed, client_id/jkt proof-of-possession) has **landed + is table-tested** (the binding-bypass the audit prioritised); audit F1 (libSQL purity wording) is resolved. **F12 (plane-bound row MAC) + F15 (schema `client_id`/`dpop_jkt` + `remote_clients` registry + `register_remote_client`/`relay_mint_remote`) are now landed too** â€” remote bearers are mintable, their binding is row-MAC-authenticated, and `relay_swap` reads it (engine + real-sqld tests). The in-process TLS+DPoP/EKM listener (F2, the only thing that *serves*, now unblocked), the bounded jti replay store (F6), streaming-revocation tear-down (F5), the PresenceGate refactor (F14), and the VPS Profile-B gates (F7â€“F9) remain â€” the SERVER-MODE audit (`docs/audits/AUDIT-server-mode.md` Â§5) is the ordered spec.

## Phase 0 â€” Scaffold that compiles
**Scope:** Cargo workspace, four members (`secrets-engine` lib, `secrets-proto` lib, `secretd` bin, `secretctl` bin), names collision-free with envctl-engine/envctl/envctl-gui; shared `[workspace.dependencies]` pinned to the **ring** crypto backend (no aws-lc-rs). Full type skeleton: `SecretEvent` + `EventSink` (std mpsc), `EngineError` + `VaultState`, the 5 seams (`Clock`/`UsbProbe`/`ProviderMint`/`Upstream`), `SecGuard` enum + `check_sec_guards` (fail-closed, `todo!()` bodies), `UnlockContext`, `RelayPolicy`/`Bearer`/`Keyslot` structs (serde-derived), a `Store` trait with the `inmem-store` impl, canonical-AAD helpers, the `clamp_ttl` choke point, `proto/control.proto` (vendored in the proto crate, referenced via `CARGO_MANIFEST_DIR`) compiling via `secrets-proto/build.rs`. Engine is a pure lib that never prints. **Destructive proto RPCs use positively-phrased `bool apply` (default false == dry-run), never `bool dry_run`.**
**Acceptance:**
- `cargo build` green on stable AND a `cargo +1.80.0 check --workspace` MSRV gate passes (idna/icu pinned if needed); `cargo clippy` clean.
- Feature matrix builds: `{default}`, `{--no-default-features --features inmem-store}`, `{inmem-store,mitm-ca}`; `compile_error!` fires on zero/both store backends.
- Dep-hygiene gates: `! cargo tree -i aws-lc-sys`, `! cargo tree | grep -E 'libsql-ffi|sqlite3-sys|openssl-sys'`, exactly one `rustls` node; `cargo tree -p envctl-secrets-engine` shows no tokio/tonic/hyper.
- A unit test asserts an all-uncertain `UnlockContext` makes EVERY `SecGuard` REFUSE (fail-closed proven before any behavior).
- A **default-constructed (all-zero) destructive request is treated as dry-run** by `check_sec_guards` (CF-8 conformance).
- One example `RelayPolicy` TOML and one `.env-ctl` profile deserialize; `SwapMode`/`Kdf` internally-tagged enums round-trip; proto `Event` <-> `SecretEvent` round-trips.
- `clamp_ttl(now, policy_ttl, requested_ttl)` test: for arbitrary inputs, `result - now <= 86400` and `<=0` ttl is refused (FS-S3 at the function level).

## Phase 1 â€” Vault core (AEAD + keyslots + canonical AAD)
**Scope:** the operator-ruled store (OI-1) behind the `Store` trait (tables: meta, keyslots, dek_generation, secrets, secret_versions, audit_log, ca_key, certs, relay_policies, relay_bearers); XChaCha20-Poly1305 per record (`chacha20poly1305`, XNonce from `OsRng`, **canonical fixed-width identity-binding AAD recomputed at decrypt, never read from a stored column**); LUKS dual-keyslot DEK wrap (USB-KEK via HKDF with domain-separated info, passphrase-KEK via `Argon2id/V0x13`); **`keyslot_aad` binds all KDF params/salt/UUID**; DEK-authenticated **vault header MAC** over the keyslot set; zeroize + fixed-capacity buffers on all key material; hash-chained audit (`append_audit`/`verify_audit_chain`), written durably. DEK rotation = **full re-encryption under one resumable atomic txn**. NO C deps.
**Acceptance:** round-trip put/get through AEAD; tamper one ciphertext byte -> decrypt fails closed (not a panic); flip any keyslot metadata byte -> `unwrap_dek` returns `None` (HF-3); a vault sealed with both slots opens with EITHER factor; `VaultEncryptedAtRest` passes only on a genuine ciphertext store; FS-S2 negative test (no plaintext bytes of a known secret in the DB file); audit-chain tamper detected at the right `seq`; a moved ciphertext+identity cannot decrypt under a different identity (HF-2 collision test); DEK rotation re-seals every row and survives a simulated mid-rotation crash (resumes via `rotation_in_progress`).
**Status (store backend, OI-1 (a)) â€” DONE:** the libSQL `remote` `Store` is implemented and **wired into `secretd`**, runtime-selected by config (`docs/ops/08-secretd-store-config.md`), and proven against a real `sqld` (5 integration tests + an engine-over-libSQL durability e2e: a secret persists across engine instances). Gate-clean (plaintext loopback connector â€” no 2nd rustls; reconnect-on-Hrana-`STREAM_EXPIRED`). The engine default store stays `inmem-store` until libSQL is configured.

## Phase 2 â€” Keymgmt / unlock (USB possession + passphrase + zeroize)
**Scope:** detect USB by **PARTUUID** (`blkid -o value -s PARTUUID` + re-verify, reusing envctl guard.rs idiom) AND **prove keyfile possession** (keyfile must unwrap the USB keyslot or match a vault-resident keyed MAC); passphrase fallback via argon2id with **entropy enforcement at enroll**; resolve-once `UnlockContext`; `UsbPresent`/`PeerIsOwner` real bodies; presence poller + `PresenceGate`; keyslot enroll/rotate-passphrase/revoke (dry-run default, header-MAC re-verified); optional **require-both** keyslot mode; `mlockall`+`RLIMIT_CORE=0`+`MADV_DONTDUMP` at daemon start (refuse to start if `mlockall` fails); KEK scratch zeroization; auto-relock-on-USB-pull DEFAULT-ON with drain grace.
**Acceptance:** USB present + keyfile valid => auto-unlock no prompt; USB absent => passphrase path; **a forged-UUID stick with the wrong keyfile does NOT pass the gate** (CF-4); unlock key never hits disk (no temp file) and is zeroized after use; single generic `UnlockFailed` regardless of which slot failed (OI-17); revoke-USB-keyslot + rekey leaves the vault passphrase-openable; `UsbPresent` REFUSES (not panics) when the identifier is unreadable (FS-S9); a low-entropy passphrase is rejected at enroll; the daemon refuses to start if `mlockall` fails (FS-S4 hardening); pulling USB zeroizes the DEK after the grace window.

## Phase 3 â€” Broker / relay (24h rotation + USB gating + adapters)
**Scope:** named policies + ephemeral tokens; bearer mint via the single `clamp_ttl` choke point with `expires_at <= now+24h`; **USB-gated AND keyfile-possession-proven** issuance/renewal; bearer **bound to policy/expiry/client_uid** (signed/MAC'd or row-by-value with asserted `policy_id` + uid); per-request `decide()` (host/path/method allowlist, rate/quota, enabled/revoked, **bearer-level revoked**, **USB re-check with grace**) â€” PURE default-deny; the three swap modes; **provider-bound canonical upstream host allowlist**; per-client audit with `token_id`. Daemon-side proxy core wired to the sync `Broker` via `relay_swap` (default-deny by construction). Per-bearer revocation RPC + `Relay.RevokeBearer`.
**Acceptance:** every minted bearer has `expires_at <= now+24h` (FS-S3) enforced at the choke point AND re-asserted on accept; **USB absent > grace AND a swap succeeds must NEVER occur** (FS-S5, HF-6); a request outside the allowlist is denied + durably audited; the real key never appears on the client-facing wire (FS-S1/FS-S6) and never egresses to an out-of-allowlist upstream (HF-11); a bearer from policy A cannot evaluate against policy B (HF-7); revoke (policy AND single bearer) takes effect on the next request and reports the count flipped (HF-16); a forced internal error on the swap path DENIES (does not send) and audits (CF-9); `decide()` has exhaustive table tests; every swap audit row joins to a unique bearer row (OI-11); clock-rollback-then-reissue is refused (OI-6).

## Phase 4 â€” Certs / CA + MITM
**Scope:** local CA (rcgen, ring backend) init/issue/renew/revoke; **CA validity <=90d auto-renewed**; sealed CA key under the DEK, in-RAM `Issuer` mlocked + zeroized on lock; **MITM leaf minting ONLY inside the relay-gated resolver** (`LeafBackedByRelay` + `UsbPresent` for every SAN, `not_after <= min(now+24h, relay validity)`, NOT NULL `relay_id`, **keys in-RAM only, not persisted**); `Certs.Issue`/`ca_issue` REFUSE `usage='mitm_leaf'`; rustls `ResolvesServerCert` returning `None` => fail-closed handshake; per-request `decide()` on established connections; NameConstraints = exact relay-host union (defense-in-depth); trust-store wiring per tool (child-only env) under reversible Wiring; **system bundle = owned-file-only + backup + fingerprint-verified revert + `--apply --confirm`, default OFF**; upstream egress pinned to **frozen webpki-roots** (never local CA, never OS store); SNI->leaf LRU cache re-checking the gate on hit.
**Acceptance:** a leaf is minted ONLY for an SNI with a currently-valid covering USB-gated relay (FS-S6 refuses otherwise, including via `ca issue` â€” CF-5); upstream TLS rejects a chain to the local CA (FS-S7, HF-12); trust wiring apply/revert round-trips with backup-before-clobber; system-bundle apply-then-revert leaves `/etc/ssl/certs/ca-certificates.crt` byte-identical to its backup minus exactly the env-ctl anchor (FS-S11, CF-7); a `git`/`gh` call through `HTTPS_PROXY` reaches upstream with the real key swapped in; USB absent => `resolve()` returns `None` even when a live bearer covers the SNI (HF-10); revoke mid-connection => the next request on the same connection is denied (OI-12); SNIâ‰ inner-Host is refused (OI-12).

## Phase 5 â€” gRPC daemon (UDS + peercred)
**Scope:** `secretd` serving the control plane over tonic on a `UnixListener` under `$XDG_RUNTIME_DIR/env-ctl` (0700 dir, 0600 sock); `SO_PEERCRED` interceptor (uid == owner, fail-closed); engine stays the pure lib; daemon installs the **ring `CryptoProvider`** explicitly; daemon drains `SecretEvent` -> gRPC server-stream + on-disk replayable log under `~/.local/state/env-ctl` (cosmetic, best-effort) while **security audit rows are committed synchronously before the RPC returns**; tracing-subscriber owned by the daemon; **proto secret/passphrase fields zeroized immediately** into engine `Zeroizing`; high-sensitivity ops require explicit unlock interaction (OI-22).
**Acceptance:** a control request from uid != owner is rejected (FS-S8, via a second uid or mocked peercred); no TCP control listener exists; socket perms verified at startup -> `EngineError::RuntimeDir` if wrong; daemon survives a client panic; the audit log is replayable; a crash between egress and event-drain leaves NO committed swap without an audit row (HF-14 crash-injection test); the upstream client refuses a local-CA chain at startup (HF-12 guard).

## Phase 6 â€” Inject / run
**Scope:** `env-ctl run -- <cmd>` exec-wrapper mints an ephemeral (or named) relay **bound to the child pid**, injects bearer + base-URL/proxy + CA-bundle env into the CHILD ONLY; **fail-closed walk-up `.env-ctl` profile discovery** (operator-trusted roots / at-or-below cwd; named-relay attach needs confirmation, pre-exec emit is a gate); the real key never enters the child env/history/git; `secret get --reveal` is `--apply`+confirm+audited and refused for broker-only secrets.
**Acceptance:** the child `/proc/<pid>/environ` contains the bearer + base-URL/proxy and NO real key (FS-S1); the parent shell env is unchanged after `run`; a per-directory profile selects the right relay; a planted ancestor profile attaching a named relay is REFUSED without confirmation (FS-S15); on USB-absent, `run` refuses a renewable long-lived token and offers only a <=24h ephemeral that then expires; `secret get --reveal` without `--apply` is dry-run (FS-S14).
