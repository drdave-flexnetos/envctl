## Findings
- **NOTE (non-blocking, design clarity — route to architect if it matters):** the `.meta.yaml`
  reader uses `BTreeMap`, so project order is **lexicographic by key, not source-file declaration
  order**. The implementer flagged this; the meta-core tab and golden test depend on it. If the
  architect intended true declaration order (e.g. dep-order for core crates), Pass 2 needs a
  preserve-insertion-order YAML mode. Currently deterministic and tested — acceptable for Pass 1.
- **NOTE (non-blocking — route to implementer at Pass 2):** the launcher `command=` in the KDL is
  the bare `envctl-dashboard-pane` (PATH-resolved). The Pass-2 manifest component MUST install it
  on PATH (e.g. symlink into `~/.local/bin`) or the generated layout fails to launch panes. Tracked
  in the handoff; not a Pass-1 defect.
- **NOTE (informational):** Pass-1 deviations are sound — GUI screen is `Screen::Mesh` (avoids
  collision with the pre-existing `Screen::Dashboard` GPU view); `--name` flag is additive. No
  behavioral deviation from the plan's parity requirement.
- **NOTE (informational):** revert-on-reset is intentionally NOT wired this pass (it is the Pass-2
  manifest component's `remove`); deploy already creates backups so reset has something to restore.
