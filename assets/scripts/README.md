# Shipped scripts (wrapped, never rewritten)

The engine references these by absolute path via `kind = "shipped_script"` hooks.
The originals live here (copied verbatim from the Desktop kit); the manifest points
at their install locations. The engine spawns them with `bash <path> [args]` and
**never opens them for writing**.

| script | install location | subcommands |
|---|---|---|
| `ubuntu-boot-repair.sh` | `/usr/local/bin/ubuntu-boot-repair.sh` | `diagnose` `repair-dev` `rename-pro` `finalize` |
| `yazelix-config.sh` | `/usr/local/bin/yazelix-config.sh` | restore `settings.jsonc` (`--force`) |
| `yazelix-setup.sh` | `/usr/local/bin/yazelix-setup.sh` | the full first-login wizard |
| `yazelix-gpu-verify.sh` | `~/.local/bin/yazelix-gpu-verify.sh` | generated by the wizard; GPU smoke test |
| `autoinstall.yaml` | USB `CIDATA/user-data` | Subiquity autoinstall |

Manifest TOML files under `../../manifest/` point at these paths. `ShippedScript`
hooks get an implicit existence guard so a missing/renamed script fails safely
rather than running the wrong thing.
